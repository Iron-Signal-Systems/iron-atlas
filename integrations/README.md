# External Integrations

Replaceable versioned adapters exchange approved evidence, metadata, canonical Atlas records, findings, context, and generated artifacts with external systems.

External systems do not become Atlas sources of authorization, canonical schema, governed-change authority, or formal-acceptance history.

The default integration modes are:

- read;
- reconcile;
- generate;
- recommend;
- export; and
- deliver.

A future adapter that writes or provisions an external system requires a separately accepted boundary with least privilege, previewable differences, actor attribution, applicable approval, bounded scope, idempotency where practical, reversal where practical, validation, retry controls, and audit history.

Initial complementary targets include Zabbix, Graylog/syslog, OpenMetrics, webhooks, SIEM and security-platform context delivery, Draw.io-compatible topology generation, and approved vendor evidence sources.
