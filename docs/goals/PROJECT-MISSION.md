# Project Mission

Build a trustworthy infrastructure knowledge and change-governance platform that helps network technicians, network administrators, network-security staff, and infrastructure teams understand what exists, how it is connected, what is changing, why it is changing, what evidence supports the conclusion, and whether the resulting environment was validated and accepted.

The interface should support the work rather than become additional work.

## Purpose

Iron Atlas is an authoritative, version-controlled system for infrastructure documentation, diagrams, inventory, automated discovery, project tracking, change management, validation, preventive health analysis, and formal acceptance.

It shall maintain a governed relationship among:

- Current infrastructure state
- Imported and collected evidence
- Normalized inventory
- Firewall policy and routing behavior
- Switch ports, trunks, neighbors, VLANs, spanning tree, and ACLs
- Wireless controllers and access points
- Projects and proposed target states
- Approved changes
- Pre-change and post-change validation
- Documentation reconciliation
- Operational findings and remediation
- External monitoring delivery
- Formal acceptance records

## Leading Principle

> Document the environment. Govern the change. Validate the result. Preserve the record.
