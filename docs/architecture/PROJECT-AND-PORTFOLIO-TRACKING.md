# Project and Portfolio Tracking

## Permanent Record

The repository retains durable project charters, requirements, decisions, current/target state, diagrams, risks, milestones, change references, validation, and acceptance.

GitHub Projects may provide board, table, roadmap, assignee, status, risk, site, workstream, and target-date views. It is an active-work interface, not the sole permanent record.

## Lifecycle

```text
Proposed → Discovery → Requirements → Design → Approved → Implementation → Validation → Documentation Reconciliation → Accepted → Completed
```

## Completion

A project is not complete because equipment was installed. Inventory, current-state diagrams, operational procedures, changes, findings, validation, exceptions, and acceptance must describe the same resulting environment.
