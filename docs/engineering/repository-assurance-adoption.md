# Iron Atlas Repository Assurance Adoption

## Current status

Atlas has a historical recorded ISRAS 1.0.1 baseline, a merged but unaccepted
ISRAS 0.1.4 candidate, and an active ISRAS 0.1.8 signer-rotation candidate. The
0.1.8 candidate is not formal adoption until exact clean-clone and hosted
evidence pass and a separate signed prospective acceptance record is merged.

0.1.8 candidate adds the current governed signing key without weakening exact

## Preserved Atlas controls

ISRAS is an organization-wide assurance floor. Atlas retains every stronger
project-specific control, including immutable historical phase gates, isolated
predecessor revalidation, PostgreSQL security and concurrency testing,
authentication and OIDC campaigns, parser and evidence validation, synchronized
documentation, and canonical clean-clone acceptance.

## Claim boundary

Self-validation is not independent review. Standards adoption does not establish
production readiness, complete vendor coverage, security certification, or
acceptance of an Atlas product phase.
