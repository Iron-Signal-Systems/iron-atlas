# Secure Development Lifecycle

Material changes follow:

```text
requirement and threat context
  ↓
architecture and bounded implementation plan
  ↓
source, tests, fixtures, and documentation
  ↓
portable pull-request validation
  ↓
canonical and specialized validation
  ↓
formal acceptance
  ↓
release, deployment, operations, and recovery
```

Repository-specific requirements, architecture, validation, and acceptance
records remain authoritative.
